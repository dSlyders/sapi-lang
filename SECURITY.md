# Security Policy

## Supported Versions

Security fixes are applied to the latest public release line in this repository (`V1`).

## Reporting a Vulnerability

Please report vulnerabilities privately.

- Contact: open a private security report via GitHub Security Advisory (preferred).
- Fallback contact: open an issue only if the report is not sensitive.

When reporting, include:

- A clear description of the vulnerability
- Reproduction steps
- Potential impact
- A suggested fix, if available

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage update: within 7 days
- Fix timeline: depends on severity and complexity

Please do not publish proof-of-concept exploit details before a fix is available.
