# Contributing

Thanks for your interest in contributing to Sapi.

## Before You Start

- Read the repository `README.md`
- Check existing issues and pull requests
- Keep changes focused and small when possible

## Development Basics

- Use clear commit messages
- Keep documentation in sync with behavior changes
- For release artifacts in this repo, update only what is needed for the active release

## Pull Request Checklist

- Code or docs changes are clearly described
- Relevant docs are updated
- No unrelated files are modified
- CI/workflows pass

## Reporting Bugs

Please open a bug report and include:

- Expected behavior
- Actual behavior
- Reproduction steps
- Environment details (OS, shell, version)

## Feature Requests

Please open a feature request with:

- Problem statement
- Proposed solution
- Alternatives considered
