Sapi

Complete language reference for building APIs that compile to Rust (Axum + Tokio).

Introduction

Sapi is a high-level language for API development. A `.sapi` source file defines services, routes, data access, auth, and runtime behavior. The compiler generates Rust code and binaries suitable for production deployment.

CLI

If the file is omitted, `sapi` tries `main.sapi` in the current directory.

CommandDescription
sapi build [file.sapi]Builds release binary into `./dist` with build log.
sapi build --linuxCross-builds Linux musl static binary.
sapi run [file.sapi]Builds and runs once.
sapi watch [file.sapi]Builds, runs, and restarts on save.
sapi-setup-<version>.exePreferred Windows installer package for end users (installs CLI + PATH).
sapi install sapi --path <path>Installs a downloaded Sapi binary and updates PATH.
sapi install rust [--linux]Installs Rust/Cargo, C++ build tools, sqlx-cli, and optional Linux build prerequisites.
sapi install all [--path <path>] [--linux]Installs Sapi CLI and Rust prerequisites in one command.
sapi migrate add|run|revert|infoSQLx-compatible migration workflow.
sapi doc [file.sapi] --htmlGenerates OpenAPI and HTML docs.
sapi plugin installInstalls VS Code extension assets from CLI.
Local prerequisite: `build`, `run`, and `watch` compile Rust locally. Rust/Cargo must be available on development machines.

Create project

sapi new my-api
cd my-api

This creates a starter project with `main.sapi` and `.vscode` snippets/settings.

Service and routes

service MyApi {
  port: 3000
  swagger: true
  cors: true
  logging: "info"
}

MyApi get "/ping" {
  return "pong"
}

MyApi post "/user" {
  body json { string name, int age }
  return 201: { string message = "Created", string user = name }
}

MyApi get "/user/{id}" {
  route { int id }
  return { int userId = id }
}

Database and SQL

database DB {
  type: "postgres"
  host: env("DB_HOST")
  port: env("DB_PORT")
  user: env("DB_USER")
  pass: env("DB_PASS")
  name: "mydb"
}

sql getUser = `SELECT id, name FROM users WHERE id = {id}` : { int:id, string:name } use DB

MyApi get "/user/{id}" {
  route { int id }
  await User row = getUser({ id: id }).one()
  return { int id = row.id, string name = row.name }
}

JWT auth and guard

auth Auth {
  secret: env("JWT_SECRET")
  algorithm: "HS256"
  expiry: 3600
  claims: { int userId, string role }
}

MyApi get "/profile" {
  guard Auth
  return "Authorized"
}

MyApi delete "/admin/user/{id}" {
  guard Auth "admin"
  route { int id }
  return { string message = "Deleted" }
}

Deployment

# Build release
sapi build

# Build Linux binary from Windows
sapi build --linux

# Run binary on server
./sapi_server

The compiled binary is standalone. Rust and Sapi are not required on the production server.

Nginx example

location / {
  proxy_pass http://127.0.0.1:3000;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
}

Documentation generation

# OpenAPI JSON to stdout
sapi doc main.sapi

# HTML docs
sapi doc main.sapi --html --output docs
Tip: Use `LANGUAGE.html` as the canonical source and sync to `Distribution/docs/index.html` and `Distribution/V1/Documentation/sapi.html`.
